## Endpoint

```
GET /api/conversations/{id}/
```

## Parámetros de Path

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `id` | integer | ID de la conversación |

## Headers

```
Authorization: Token <your-token>
Content-Type: application/json
```

## Campos de Respuesta

<ResponseField name="id" type="integer">
  ID único de la conversación
</ResponseField>

<ResponseField name="contact" type="object">
  Información del contacto
</ResponseField>

<ResponseField name="whatsapp_line" type="object">
  Línea de WhatsApp asociada
</ResponseField>

<ResponseField name="status" type="string">
  Estado: `ACTIVE`, `WAITING`, `CLOSED`
</ResponseField>

<ResponseField name="assigned_agent" type="object">
  Agente asignado (puede ser null)
</ResponseField>

<ResponseField name="active_flow_execution" type="object">
  Ejecución de flujo activa (puede ser null)
</ResponseField>

## Ejemplo de Uso

<CodeGroup>

```bash cURL
curl -X GET "https://app.convey.pro/api/conversations/1/" \
  -H "Authorization: Token your-token-here"
```

```python Python
import requests

response = requests.get(
    "https://app.convey.pro/api/conversations/1/",
    headers={"Authorization": "Token your-token-here"}
)

conversation = response.json()
print(f"Conversación con: {conversation['contact']['name']}")
print(f"Status: {conversation['status']}")
print(f"Mensajes: {conversation['message_count']}")
```

</CodeGroup>

## Errores

| Código | Descripción |
|--------|-------------|
| 401 | Token no proporcionado o inválido |
| 403 | Sin permisos para ver esta conversación |
| 404 | Conversación no encontrada |
